# KiloRAG: Система автоматизированного расчета категорий по пожарной и взрывопожарной опасности

## Описание проекта

KiloRAG — это система, разработанная для автоматизированного выполнения комплексного анализа и расчета категорий помещений, зданий и наружных установок по взрывопожарной и пожарной опасности в соответствии с требованиями российского законодательства и нормативных документов.

Система основана на методиках:
- [1] Методика определения показателей пожаровзрывоопасности технологических сред
- [2] Методика определения классификационных показателей пожароопасных и взрывоопасных зон
- [3] СП 12.13130.2009 "Определение категорий помещений, зданий и наружных установок по взрывопожарной и пожарной опасности"

## Сущность методологии

Методология определения категорий помещений по пожарной и взрывопожарной опасности включает в себя систематизированный подход к оценке опасности объектов производственного и складского назначения класса Ф5. Основные аспекты методологии:

1. **Предварительное планирование и проектирование архитектуры мероприятий** - сбор исходных данных и определение перечня необходимых расчетов
2. **Анализ исходных данных** - изучение технологического процесса, перечня обращающихся веществ и материалов, параметров технологических сред, согласно статье 16 123-ФЗ
3. **Определение показателей пожаровзрывоопасности технологических сред** - расчет характеристик горючих веществ (газов, паров, пылей, аэрозолей, жидких и твердых веществ)
4. **Определение классификационных показателей пожароопасных и взрывоопасных зон** - классификация зон согласно статьям 18 и 19 123-ФЗ
5. **Расчет категорий помещений, зданий и наружных установок** - применение методики СП 12.13130.2009
6. **Оформление заключения** - подготовка экспертного заключения с рекомендациями

## Методики

### 1. Методика определения показателей пожаровзрывоопасности технологических сред

Методика устанавливает порядок расчета и анализа показателей пожаровзрывоопасности для различных типов технологических сред в соответствии с ГОСТ 12.1.044-2018. Основные классы технологических сред:
- Газообразные и парообразные вещества
- Легковоспламеняющиеся и горючие жидкости
- Горючие пыли и аэрозоли
- Твердые вещества и материалы

Для каждой категории рассчитываются соответствующие показатели:
- Концентрационные пределы воспламенения
- Температурные параметры (вспышки, воспламенения, самовоспламенения)
- Минимальная энергия зажигания
- Индекс взрывоподъемности
- Кислородный индекс

### 2. Методика определения классификационных показателей пожароопасных и взрывоопасных зон

Методика устанавливает порядок определения классификационных показателей пожароопасных и взрывоопасных зон в соответствии с Федеральным законом N 123-ФЗ. Зоны классифицируются по следующим критериям:
- П-I: Помещения с обращением горючих жидкостей с температурой вспышки 61°C и более
- П-II: Помещения с выделением горючих пылей или волокон
- П-IIа: Помещения с обращением твердых горючих веществ
- П-III: Наружные установки с обращением горючих веществ

Взрывоопасные зоны классифицируются как:
- Класс 0: Постоянно присутствующие взрывоопасные смеси
- Класс 1: Возможные взрывоопасные смеси при нормальном режиме
- Класс 2: Возможные взрывоопасные смеси только при авариях
- Класс 20, 21, 22: Для пылевых смесей

### 3. СП 12.13130.2009

Свод правил устанавливает методы определения категорий помещений, зданий и наружных установок по взрывопожарной и пожарной опасности. Категории помещений:
- **А** - Взрывоопасные смеси горючих газов или паров ЛВЖ с воздухом
- **Б** - Взрывоопасные смеси горючих пылей или волокон
- **В1-В4** - Пожароопасные помещения с разной степенью пожарной нагрузки
- **Г** - Негорючие вещества в горячем состоянии
- **Д** - Негорючие вещества в холодном состоянии

## Установка и настройка

### I. Установка npm через Windows PowerShell

1. Откройте PowerShell от имени администратора
2. Проверьте наличие установленного Node.js:
   ```powershell
   node --version
   ```
3. Если Node.js не установлен, скачайте установочный файл с официального сайта Node.js (https://nodejs.org/)
4. Запустите установочный файл и следуйте инструкциям
5. После установки Node.js npm будет установлен автоматически
6. Проверьте установку npm:
   ```powershell
   npm --version
   ```

### II. Установка Qwen-Code через npm

1. Убедитесь, что npm установлен (см. предыдущий раздел)
2. Установите Qwen-Code глобально:
   ```powershell
   npm install -g @qwen-code/qwen-code
   ```
3. Проверьте установку:
   ```powershell
   qwen
   ```

### III. Установка VSCode и KiloCode

1. Скачайте и установите Visual Studio Code с официального сайта: https://code.visualstudio.com/
2. Запустите VSCode
3. Перейдите в раздел расширений (Extensions)
4. Найдите и установите расширения `KiloCode`, `Markdown All in One`, `Markdown Editor`, `Markdown Table` и `markdownlint`
5. Перезапустите VSCode для завершения установки

---

**Примечание: Из расширений достаточно установить только `KiloCode`**

---

### IV. Установка RAG в KiloCode

1. Откройте VSCode с установленным KiloCode
2. Кликните на иконку весов Фемиды ⚖️ в боковой панели
3. Во вкладке `Rules` в разделе `global rules` напишите "Методология определения категорий.md" и нажмите Enter
4. Кликните на иконку карандаша ✏️. Откроется окно с пустым файлом
5. Скопируйте содержимое файла [`БАЗА/ ЗНАНИЙ/ДЛЯ ОТЛАДКИ/mcp-rules/Методология\ определения\ категорий.md`](БАЗА/ ЗНАНИЙ/ДЛЯ ОТЛАДКИ/mcp-rules/Методология\ определения\ категорий.md:1) в открывшийся пустой файл и сохраните
6. Система готова к работе

## Использование системы

### Алгоритм действий:

1. Загрузите исходные данные в формате docx, pdf, txt, md, xml, html, tex, doc в папку "исходные данные"
2. Отредактируйте файл [`промт.txt`](промт.txt:1) в соответствии с исходными данными, которые вы предоставляете модели для анализа
3. В текстовом окне чата KiloCode поставьте символ @, введите "промт", нажмите `Tab` на клавиатуре и Enter

Модель выполнит анализ, произведёт расчеты и создаст в корневом каталоге технический отчет в формате `.md` с сопутствующими техническими заключениями.

С помощью онлайн сервисов вы можете конвертировать файлы формата MD в формат DOCX или PDF.
